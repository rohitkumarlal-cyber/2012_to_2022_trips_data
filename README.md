#  Cycle Trips Data Analysis (2013–2022)

##  Project Overview

This project analyzes bike-sharing trip data from **2013 to 2022** to understand user behavior, trip patterns, and system usage trends.
The goal of this project is to demonstrate a **complete analytics workflow** using:

* **Python** for data cleaning
* **PostgreSQL** for data storage and transformation
* **SQL Views** for analytical queries
* **Power BI** for interactive dashboard visualization

The final result is an interactive dashboard that highlights **trip trends, user types, popular routes, and seasonal patterns** in bike usage.

---

## Tech Stack

* **Python (Pandas, NumPy)** – Data cleaning and preprocessing
* **PostgreSQL** – Database management and analytical queries
* **SQL** – Data transformation using views
* **Power BI** – Data visualization and dashboard creation
* **GitHub** – Project documentation and version control

---

## Dashboard Insights

The Power BI dashboard provides insights into several aspects of bike usage:

### 1️⃣ Trip Usage by Year

Shows how bike trip usage has changed from **2013 to 2022**, highlighting overall growth and fluctuations in system usage.

### 2️⃣ User Type Distribution

Breakdown of trips by user category:

* **Member**
* **Casual**
* **Dependent**

This helps understand which type of users generate the most activity.

### 3️⃣ Average Ride Duration

Compares average ride times between different user types to analyze behavioral differences.

### 4️⃣ Busiest Days of the Week

Identifies which days experience the highest ride volumes.

### 5️⃣ Busiest Months

Shows seasonal trends in bike usage and highlights peak demand periods.

### 6️⃣ Most Common Routes

Displays the **top 10 most frequently used station-to-station routes**.

---

## 🔍 Key Insights

* **Members generate the majority of bike trip**
* **Cycling activity peaks during summer months, with July and August showing the highest trip counts**
* **Saturdays record the highest trip volume, indicating strong weekend usage**
